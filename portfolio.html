A General Assemblage of Trivia

The project is an implementation of a multiple choice trivia game. There's only four questions included, but since they're stored as a JavaScript hash, more questions can easily be added.

The implementation is in JavaScript and JQuery, alongside HTML and CSS.

To summarize how the project works, the questions are stored as a hash, consisting of

  1. the question itself
  2. a list of answers, with one having a "correct" flag
  3. a score for the question
  4. an optional image to be displayed with the question

The program goes through the list of questions in order. The order of answers is randomized each time, so the first answer one time might be the third another time.

It listens for a click on one of the answers. If the answer is correct, the score for the question is added to the user's score. If it's wrong, one point is subtracted, though the score cannot drop below zero.

Once the code reaches the last question, it repeats that question perpetually.

This could easily be made into a more serious quiz. More questions could be added.

A large increase in the number of questions would require a database of questions. I'd like to add that in the future, and also add things like allowing the user to select categories and difficulty. Beyond that, a database would allow persistent scoring with user logins.

Still more details can be found in the project's README file.

Together, They're Detectives!

One of our assignments for the course was to explore an API of our choosing. One of our instructors suggested the API for INeedAPrompt, a website he created to generate random characters and situations.

I explored the site and its API, and found it amused me greatly. It reminded me of an old joke that you can take descriptions of two utterly bizarre characters, add "togethet, they're detectives!", and it creates the plot of a TV show. (TVTropes calls the concept They Fight Crime.

The interface is very simple. There are ten pull-downs, each allowing the user to select one of the five types of items (adjective, noun, adverb, verb, location) the API returns.

Each time a selector is changed, the "plot description" in the top box regenerates. I realize that, from a user interface perspective, the automatic regeneration is annoying, to say the least. From a technical standpoint, though, getting it to work was a nice challenge. I would like to add a button that will allow the user to just press a seprate button to regenerate.

Eventually, I would like to fork the INeedAPrompt source code, strip it down to just the core that generates the prompts, and rewrite it to male it my own. This serves several purposes:

  1. It allows me to keep working on, and learning from, a project that has brought my amusement. I'll probably convert it to a different language.
  2. I won't be dedendent on a web site maintained by someone else to keep my project working.
  3. I will be able to host the application at a secure site. When I attempted to host it in my github.io space, I was unable to connect to the API, since it is hosted at a non-secure site.

This is currently not hosted anywhere, due to the issues with connecting to the API mentioned above.

To view it, download the source from the project github repo, and view index.html in a browser.
Personal Web Site

This site, in other words. It's all HTML, CSS and a little JavaScript/JQuery. The scripting covers one thing -- this is all done as one page, with the menu executing a script that changes the body of the page.

This demonstrates my philosophy of web design. While I am much more of a developer than a designer, I prefer web sites to be simple, even spartan. This is a personal bias, dating back to a time when I was using a dial-up connection on a somewhat defective modem. Overly complex sites were slow to load, and I came to prefer simplicity. I think craigslist and The Drudge Report are good style models.

That aside, I do have an engineer's curiosity and desire to learn, so I've added a few "Easter Eggs". For instance, try hovering the mouse pointer over my portrait in the upper right. I'll add more later.
